什么是hash表呢？

Leetcode 哈希表
算法之从入门到放弃
发布日期:   2020-07-07

哈希表！？

大家在第一次听到哈希表的时候，会怎么去想这个东西呢？反正我是觉得挺神秘的！
一定是个很厉害的算法，其实不是，是一个很菜鸟的算法！有多菜呢？我都看得懂！
哈哈哈，那我们就来一起看看什么是哈希表吧！

哈希表简述

散列表（Hash table，也叫哈希表），是根据关键码值(Key value)而直接进行访问的数据结构。也就是说，
它通过把关键码值映射到表中一个位置来访问记录，以加快查找的速度。这个映射函数叫做散列函数，
存放记录的数组叫做散列表。给定表M，存在函数f(key)，对任意给定的关键字值key，代入函数后若
能得到包含该关键字的记录在表中的地址，则称表M为哈希(Hash）表，函数f(key)为哈希(Hash) 函数。

我们给数据找个代号，然后直接用代号来访问这个数据，这个算法就对应了c++里面的map和unordered_map

map和unordered_map的区别

引入头文件不同

类别 头文件
map #include < map >
unordered_map #include < unordered_map >

实现机制不同

类别 实现机制
map 红黑树
unordered_map hash表

优缺点比较

类别 优点 缺点
map 有序性，简化操作 空间占用率高
unordered_map 内部哈希表，查找快 建立比较耗费时间

两数之和（leetcode）从题目认识哈希

题目阐述

给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。
你可以假设每种输入只会对应一个答案。但是，数组中同一个元素不能使用两遍。

两次哈希

对于第一种暴力破解，相当于遍历了所有的两数之和的情况，如果满足就输入，这是很浪费时间的。
我们使用哈希来优化：
1.unordered_map内部显示机制是hash表，更贴合算法，我们使用unordered_map
2.

graph TD;
    数据输入->压入unordered_map
    压入unordered_map->循环map里面的值i
    循环map里面的值i->条件值为target-nums
    条件值为target-nums->查找满足条件值的下标j
    查找满足条件值的下标j--得到结果->输出i和j
    查找满足条件值的下标j--未找到则i++->循环map里面的值i

一次哈希

和两次哈希不同的是，在压入的同时完成了查询。
那这样会不会漏掉情况呢？是不会的！
就好比力的作用是相互的，后面出现了能和前面匹配的数据，前面没找到后面，但是后面可以找到前面的数字。

三种解法的时间空间复

什么是时间复杂度？

：时间复杂度的计算并不是计算程序集体运行的时间，二十算法执行语句的次数

什么是空间复杂度？

：空间复杂度是对一个算法在运行过程中临时占用储存空间大小的量度

三种解法复杂度分析

：
时间复杂度：O(n^2)
两层for循环，对于每个元素，耗费O(n)的时间，所以时间复杂度为O(n^2)
空间复杂度：O(1)
：
时间复杂度：O(n)
我们把包含有n个元素的列表遍历两次，由于哈希表将查找时间缩短到O(1)，所以时间复杂度为O(n)
空间复杂度：O(n)
所需的额外空间取决与哈希表中存储的元素数据，该表中存储了n个元素
：
时间复杂度：O(n)
我们把包含有n个元素的列表遍历一次，由于哈希表将查找时间缩短到O(1)，所以时间复杂度为O(n)
空间复杂度：O(n)
所需的额外空间取决与哈希表中存储的元素数据，该表最多需要存储n个元素

算法 时间复杂度 空间复杂度
暴力破解 O(n^2) O(1)
二次哈希 O(n) O(n)
一次哈希 O(n) O(n)

遇到的问题

在开始提交的时候没有通过所有样例

原因是没有添加判断，
引用具有指定键的元素的位置的迭代器，如果找不到具有键的匹配项，则引用映射中最后一个元素后面的位置。
即m.end()，我们需要判断是否查询到值。
